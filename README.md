# Relayout Editor 🎨

Chrome Extension สำหรับทีม UI/UX — เปิดหน้าเว็บจริงแล้วแก้ layout ได้ทันที
ลากย้ายเป็น block · ปรับขนาด · เปลี่ยนสี · เปลี่ยนรูป · แก้ข้อความ · เพิ่ม element ใหม่
งานบันทึกอัตโนมัติข้าม refresh แล้ว **export** เป็นรายงาน / CSS / ภาพ ส่งให้ dev ได้เลย

## วิธีติดตั้ง

1. เปิด Chrome ไปที่ `chrome://extensions`
2. เปิดสวิตช์ **Developer mode** (มุมขวาบน)
3. กด **Load unpacked** แล้วเลือกโฟลเดอร์นี้ (`ex-relayout`)
4. ปักหมุดไอคอน Relayout Editor ไว้ที่ toolbar

## วิธีใช้

เปิดหน้าเว็บที่ต้องการแก้ แล้ว**คลิกไอคอน extension** → panel จะโผล่ขวาบน (คลิกซ้ำ = ปิด · ปุ่ม ▁ = ย่อ panel เหลือแค่แถบหัว · ลากแถบหัวเพื่อย้ายที่)

Panel แบ่งเป็น **4 แท็บ**:

| แท็บ | มีอะไร |
|---|---|
| **แก้ไข** | selector + breadcrumb, ปุ่มจัดการ (ข้อความ/ขึ้น-ลง/คัดลอก/ทำซ้ำ/ซ่อน/ลบ/รีเซ็ต) และ inspector แบบหมวดพับได้ (สี & ตัวอักษร · ขนาด & ระยะ · เส้นขอบ & เอฟเฟกต์ · Layout/Grid · CSS กำหนดเอง) — เลือกช่องตาราง/รูป จะมีหมวดคอลัมน์ตาราง/รูปภาพโผล่ให้เอง |
| **เพิ่ม** | element ใหม่ (กล่อง/รูป/ปุ่ม/หัวข้อ/ข้อความ/วาง ⌘V), section, และคลัง Asset |
| **รายการ** | รายการแก้ไขทั้งหมด (มี badge นับจำนวนบนแท็บ) + รีเซ็ตทั้งหน้า |
| **ส่งออก** | รายงาน JSON / CSS / HTML, screenshot, ขนาดจอ responsive, import JSON |

คลิกเลือก element บนหน้าเมื่อไหร่ panel จะพากลับมาแท็บ **แก้ไข** ให้อัตโนมัติ

### โหมดการทำงาน (ปุ่มบนสุดของ panel)

- **✏️ โหมดแก้ไข** — cursor เป็น crosshair, คลิก = เลือก element (ค่าเริ่มต้น)
- **🖱 โหมดใช้งาน** — cursor ปกติ, คลิกลิงก์/ปุ่ม/ฟอร์มของหน้าได้จริง (ไว้ทดลอง flow ระหว่างแก้ โดยงานที่แก้ค้างไว้ไม่หาย)

### เลือกและแก้ element

| การกระทำ | วิธี |
|---|---|
| เลือก element | คลิกที่ element |
| เลือกตัวแม่ | คลิก breadcrumb ใต้ selector (ไต่ขึ้นได้ทีละชั้น) |
| ย้ายตำแหน่ง (block) | เลือกแล้ว**ลาก** — ตัว element ลอยตามเมาส์ เส้นม่วง = แทรกก่อน/หลัง, **กรอบม่วงเส้นประ** (ลากไปกลาง container) = ยัดเข้าไปข้างในกล่อง/section |
| เลือกหลายชิ้น (คลุม) | **Shift+ลากคลุม** element ที่อยู่ในกรอบทั้งตัว · **Shift+คลิก** = เพิ่ม/เอาออกทีละชิ้น · ลากชิ้นไหนก็ได้ในกลุ่ม = **ย้ายทั้งกลุ่มพร้อมกัน** (ลำดับคงเดิม, undo เดียว) · Delete = ซ่อนทั้งกลุ่ม · Esc = ยกเลิกกลุ่ม |
| ปรับขนาด | ลาก**จุดกลม**ที่ขอบขวา/ล่าง/มุมของกรอบเลือก (หรือพิมพ์เลขใน panel) |
| สลับลำดับใน DOM | ปุ่ม ⬆︎ ย้ายขึ้น / ⬇︎ ย้ายลง |
| สลับคอลัมน์ตาราง | เลือกช่องในตาราง (th/td) → ปุ่ม ⬅/➡ ย้ายคอลัมน์ — ทุกแถวสลับตาม (ตารางที่มี colspan/rowspan จะไม่ให้สลับ) |
| สไตล์ | สีพื้น/สีตัวอักษร/ฟอนต์/มุมโค้ง/padding/กว้าง/สูง/**ตัวหนา/จัดชิด/เส้นขอบ/เงา/ความทึบ** |
| Margin | ช่องแยก 4 ด้าน (บน/ขวา/ล่าง/ซ้าย) |
| จัด Layout / Grid | เลือกโหมด Block / Flex แถว / Flex คอลัมน์ / **Grid** + จำนวนคอลัมน์ + gap — ใช้กับ section/กล่องที่สร้างใหม่แล้วลาก element เข้าช่องได้เลย |
| CSS กำหนดเอง | textarea พิมพ์ CSS ตรง ๆ (`prop: value;` หลายบรรทัด) กด ✨ ใช้ — ตรวจ syntax ให้, undo ได้ทั้งชุด, ออกใน export ทุกแบบ |
| แก้ข้อความ | **ดับเบิลคลิก** (หรือปุ่ม ✏️) เสร็จแล้วกด Esc |
| เปลี่ยนรูป | เลือก `<img>` แล้ววาง URL หรือเลือกไฟล์จากเครื่อง |
| คัดลอก / วาง | ปุ่ม 📄/📋 หรือ **⌘C** / **⌘V** — วางซ้ำได้หลายครั้ง |
| ทำซ้ำ | ปุ่ม ⧉ — สำเนาต่อท้ายทันที |
| ซ่อน | ปุ่ม 🙈 หรือ **Delete** (รายงานบอก dev ว่า "ซ่อนไว้") |
| ลบจริง | ปุ่ม 🗑 หรือ **Shift+Delete** (รายงานบอกว่า "เอาออกเลย") |
| Undo / Redo | ↩︎ / ↪︎ หรือ **⌘Z** / **⇧⌘Z** |
| รีเซ็ต element เดียว | ปุ่ม ♻️ รีเซ็ต (หรือ ♻️ ในรายการแก้ไข) |
| รีเซ็ตทั้งหน้า | ปุ่ม ♻️ รีเซ็ตทั้งหน้า — คืนทุกอย่างรวมถึงตำแหน่ง DOM เดิม |

### เพิ่มของใหม่

ทุกอย่างวางต่อจาก element ที่เลือก (ไม่ได้เลือก = ต่อท้ายหน้า) แล้วแก้/ลาก/ลบต่อได้เหมือน element ทั่วไป

> **เส้นประรอบกล่อง/section ใหม่เป็นแค่ไกด์ตอนแก้** — สลับเป็นโหมดใช้งาน, ถ่าย screenshot, หรือ export เมื่อไหร่จะไม่มีเส้นประติดไปด้วย ไม่ต้องลบเอง

- 🔲 **กล่อง** เปล่า (ลาก element อื่นยัดเข้าไปได้)
- 🖼️ **รูปภาพ** placeholder → เลือกรูปแล้วเปลี่ยน URL/ไฟล์
- 🔘 **ปุ่ม** · 𝗛 **หัวข้อ** · ¶ **ข้อความ**
- ➕ **Section** (ก่อน/หลังที่เลือก หรือต่อท้ายหน้า)

### คลัง Asset (ใช้ข้ามหน้า / ข้ามเว็บ)

- เลือก element แล้วกด **📦 เก็บเข้าคลัง** (ตั้งชื่อได้) — ระบบอบ computed styles ลง inline ให้ด้วย จึงหน้าตาใกล้เคียงเดิมแม้วางบนหน้าที่ไม่มี CSS ของหน้าต้นทาง
- คลังอยู่ใน `chrome.storage` ของ extension → เปิด editor บน**หน้าไหนก็ได้**แล้วกด 📌 วางลงหน้านั้น (เก็บล่าสุด 30 ชิ้น, 🗑 ลบออกจากคลังได้)

### รายการแก้ไข + งานต่อเนื่อง

- **รายการแก้ไข** ใน panel — คลิกชื่อเพื่อกระโดดไปดู element, กด ♻️ รีเซ็ตรายตัว
- **บันทึกอัตโนมัติ** — ทุกการแก้ไขเซฟลง `chrome.storage` ต่อ URL, refresh แล้วเปิด editor ใหม่จะมีแถบ **"พบงานที่ค้างไว้ → กู้คืน/ทิ้ง"**
- ถ้าหน้าเป็น SPA แล้ว framework render ทับของที่แก้ จะมี toast เตือน (กู้คืนจาก storage หรือ import ได้)

### ขนาดจอ (ทดสอบ Responsive)

ปุ่ม 📱 390 / 📱 768 / 💻 1280 ปรับความกว้างหน้าต่างเบราว์เซอร์ให้ viewport ≈ ค่านั้น (media query ทำงานจริง) · ↩︎ คืนเดิม

### Export ส่งให้ Dev

- **📋 รายงาน JSON** — ทุกการแก้ไข: selector, สไตล์ `from → to`, ข้อความ/รูป, ตำแหน่งใหม่ที่ย้ายไป, ซ่อน/ลบ, element ที่เพิ่ม (พร้อม HTML)
- **🎨 CSS** — สไตล์ที่แก้เป็น `selector { prop: value }` ก็อปวางได้เลย
- **🧾 HTML** — snapshot ทั้งหน้าหลังแก้ (`data-rl-changed` / `data-rl-added` ติดอยู่)
- **📸 จอ / ที่เลือก / ทั้งหน้า** — ภาพ PNG: viewport ปัจจุบัน, เฉพาะ element ที่เลือก, หรือทั้งหน้าแบบต่อภาพ (จำกัด ~8000px)
- **📥 Import JSON** — โหลดรายงานกลับมาแก้ต่อ/รีวิวบนหน้าจริง (ส่งไฟล์ให้เพื่อนร่วมทีมได้)

## โครงสร้างไฟล์

- `manifest.json` — Manifest V3, สิทธิ์ `activeTab` + `scripting` + `storage`
- `background.js` — service worker: inject editor, ถ่าย screenshot, ปรับขนาดหน้าต่าง (responsive preview)
- `editor.js` — ตัว editor ทั้งหมด (UI อยู่ใน Shadow DOM จึงไม่ชนสไตล์ของหน้าเว็บ)
- `test-page.html` — หน้าตัวอย่างไว้ลองเล่น (เปิดด้วย `open test-page.html`)

## ข้อจำกัด

- ใช้กับหน้า `chrome://`, Chrome Web Store, หรือ PDF viewer ไม่ได้ (Chrome ห้าม inject)
- การกู้คืน/Import อ้าง element ด้วย CSS selector — ถ้าโครงหน้าเปลี่ยนไปมาก บางรายการอาจหา element ไม่เจอ (จะแจ้งจำนวนที่พลาด)
- Screenshot ทั้งหน้า: ส่วนที่เป็น `position: fixed` (header ลอย) จะติดซ้ำในแต่ละช่วงภาพ
